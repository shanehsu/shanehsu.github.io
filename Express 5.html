Express 5

Express 是一個非常強大的 HTTP 伺服器套件。

Express 套件

const express = require('express')

express 本身有提供兩個功能，express.static() 以及 express.Router()。

express.static() 是一個用來提供靜態網頁的中介軟體。
express.Router() 會回傳一個可使用的 Router 實例。

應用程式（Application） 物件

const app = express()

屬性

app.locals

一個物件，裡面的值由整個應用程式共享。

app.locals.title = 'An Express Application' // 設定
const title = app.locals.title              // 使用

app.mountpath

只可讀取。
若透過子程式（另外一個 Application）來進行路由，子程式的 mountpath 就是他被裝載的位置。

const subapp = express()
app.use('/subapp', subapp)
subapp.mountpath // '/subapp'

事件

admin.on('mount', callback)

admin.on('mount', parent => {
  console.log('Admin Mounted')
  console.log(parent)
})

方法

app.all(path, callback)

這一個路由將會比對所有 HTTP 動作。

app.all('*', (req, res, next) => {
  let authenticated = false
  // 認證
  if (authenticated) next()
  else next(new Error('Not Authenticated'))
})

app.disable(name)
app.disabled(name)
app.enable(name)
app.enabled(name)

與設定有關的方法。他們將設定值設定為 true 或是 false。

app.engine(ext, callback)

設定樣版引擎。

app.set(name, value)
app.get(name)

取得、設定設定值。

app.listen(port, hostname, backlog, callback)

通常只需要傳入 port。

app.get(path, callback)
app.post(path, callback)

這一群有關設定路由的方法，可用的動詞包含

  • checkout
  • connect
  • copy
  • delete
  • get
  • head
  • lock
  • merge
  • mkactivity
  • mkcol
  • move
  • m-search
  • notify
  • options
  • patch
  • post
  • propfind
  • proppatch
  • purge
  • put
  • report
  • search
  • subscribe
  • trace
  • unlock
  • unsubscribe

app.param(name, callback)

當遇到名為 name 的路由參數，會先經過 callback。

app.param('user', (req, res, next, id) => {
  User.find(id, function(err, user) {
    if (err) {
      next(err)
    } else if (user) {
      req.user = user
      next();
    } else {
      next(new Error('failed to load user'))
    }
  })
})
app.get('/users/:user', (req, res, next) => {
  res.json(req.user)
})

app.path()

回傳該 app 的路由路徑。

app.render(view, locals, callback)

利用所設定的樣版引擎處理 HTML 畫面。

app.route(path)

回傳一個路徑，可以進一步的設定路由。

var app = express();
app.route('/events')
.all((req, res, next) => {
  
})
.get((req, res, next) => {
  res.json({})
})
.post((req, res, next) => {
  
})

app.use(path, middleware)

載入中介軟體到指定路由。若省略 path 則每個要求都會經過該中介軟體。

app.use(express.static(__dirname + '/public'))
app.use('/static', express.static(__dirname + '/public'))

可用的設定值

  • case sensitive routing
  • env
  • etag
  • jsonp callback name
  • json replacer
  • json spaces
  • query parser
  • strict routing
  • subdomain offset
  • trust proxy
  • views
  • view cache
  • view engine
  • x-powered-by

要求（Request）物件

屬性

req.app

該要求所屬的應用程式。

req.baseUrl

該要求的路由網址（不包含要求的最後一段路由）

const greet = express.Router()
greet.get('/jp', (req, res) => {
  console.log(req.baseUrl) // /greet
  res.send('Konichiwa!')
})
app.use('/greet', greet)

req.body

所使用 body-parser 或是 multer 中介軟體，將會把要求主體存入該屬性。

若有一個 PATCH 其主體是 JSON

app.patch('/resources/:id', (req, res, next) => {
  const id = req.params.id
  const delta = req.body
  
  Resource.findByIdAndUpdate(id, delta).then(() => {
    res.sendStatus(200)
  }).catch(() => next(new Error('無法更新指定資源')))
})

req.cookies

若使用 cookie-parser 中介軟體，將會把要求的 Cookie 存入該屬性。

req.hostname

主機位址。

req.ip

用戶端位址。

req.ips

經過的 IP 位址（包含代理伺服器）。

req.method

該要求的 HTTP 動詞。

req.originalUrl

該要求的原始網址。

req.params

該要求的路由參數。

req.path

該要求的路由路徑。

// example.com/users?sort=desc

req.path

// => "/users"

req.protocol

http 或是 https

req.query

該要求的 query 資料。

// GET /search?q=tobi+ferret
req.query.q // => "tobi ferret"
// GET /shoes?order=desc&shoe[color]=blue&shoe[type]=converse
req.query.order // => "desc"
req.query.shoe.color // => "blue"
req.query.shoe.type // => "converse"

req.route

傳回有關此路由的資訊，包含完整路徑以及處理常式。

{
  path: '/user/:id?',
  stack: [
    { 
      handle: [Function: userIdHandler],
      name: 'userIdHandler',
      params: undefined,
      path: undefined,
      keys: [],
      regexp: /^\/?$/i,
      method: 'get'
    }
  ],
  methods: {
    get: true
  }
}

req.secure

傳回該要求是否透過 https。

req.signedCookies

當使用 cookie-parser 時，signedCookies 與 cookies 相同，不過 cookie-parser 會將有保護的 cookie 解碼病存到這裡來。

req.subdomains

// 主機是 tobi.ferrets.example.com
req.subdomains // => ["ferrets", "tobi"]

方法

req.accepts(types)
檢查哪一種型態可以被回傳

// Accept: text/*, application/json
req.accepts('image/png')
req.accepts('png')
// => undefined

req.acceptsCharsets(charset [, ...])
req.acceptsEncodings(encoding [, ...])
req.acceptsLanguages(lang [, ...])

與上面的相似，分別檢查不同標頭。

req.get(field)

取得標頭的某一欄位。

req.is(type)

詢問要求的種類。

// 要求是 application/json 時
req.is('json')
req.is('application/json')
req.is('application/*')
// => true

回應物件

屬性

res.app

回傳所屬的應用程式實例。

res.headersSent

回傳是否已經傳送了該回應的標頭。
當值為 true 基本上代表要求已經被回應。

res.locals

類似 app.locals，可以儲存有關該要求、回應的相關資料。

方法

res.append(field, value)

value 可以省略，但若存在，必須是字串或是陣列。

res.attachment(filename)

當沒有提供 filename 時，將 Content-Disposition 設為 attachment；當提供 filename 時，會將 Content-Disposition 以及 Content-Type 設定好。

res.cookie(name, value, option

寫入 Cookie。option 是一個設定物件，可以接受的設定鍵值，

  • domain
  • encode
  • expires
  • httpOnly
  • maxAge
  • path
  • secure
  • signed

res.clearCookie(name, option)

與 res.cookie 功能相反。

res.download(path, filename, error_callback)

提供 path 所指定的檔案，可以指定下載下來的檔案名稱（filename) 以及當有錯誤的時候的處理常式。

res.end()

快速傳回空資料。

res.status(404).end()

res.format(object)

讓 Express 根據 Accepts 標頭選擇最適合的回應。

res.format({
  'text/plain': () => {
    res.send('hey')
  },
  'text/html': () => {
    res.send('<p>hey</p>')
  },
  'application/json': () => {
    res.send({ message: 'hey' })
  },
  'default': () => {
    res.status(406).send('Not Acceptable')
  }
})

也可以使用像是 json、html 等通用名稱。

res.get(field)

取得標頭的某一個欄位值。

res.json(body)
res.jsonp(body)

以 JSON 或是 JSONP 方式回傳一個物件。

JSONP 的預設處理常式名稱會依照網址的 Query callback，可以透過更改應用程式設定來修改。

res.links(links)

其中 links 是一個物件，會修改 HTTP 的 Links 標頭。

Links 標頭所指向的網址通常帶有相關資料。

res.location(path)

設定 Location 標頭。

res.redirect(status, path)
res.redirect(path)

重新導向使用者。

res.render(view, locals, callback)

與 app.render 相似，但是會送出處理後的結果。

res.send(body)

送出 body 指定的內容，若不提供 body 則還是可以送出回應。若 body 為 Buffer 則 Content-Type 將自動被設定為 octet-stream（二進位），除非 Content-Type 已經被定義。若送出字串，Content-Type 會被設定為 text/html；若送出陣列或是物件，將會被轉換為 JSON。

res.sendFile(path, options, callback)

送出一個檔案，可以設定的選項有

  • maxAge
  • root
  • lastModified
  • headers
  • dotfiles

提供錯誤處理常式。

res.sendStatus(statusCode)

將會送出一段解釋文字。

一個 sendStatus 等義於

res.status(200).send('OK')

res.set(field, value)

設定標頭欄位值。

res.status(code)

設定狀態碼。

res.type(type)

設定 Content-Type。

res.vary(field)

將 field 加入 Vary 標頭中。

Vary 的功能是告訴做出要求的人，當某個 field（某一個標頭欄位）被更改時，會有不同的回應。

假設 Vary: Accept-Language，那就代表，我們的網頁會有多種語言，當你改變你所接受的語言，應該重新送出要求，以獲得最適合的資訊。
