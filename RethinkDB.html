RethinkDB

RethinkDB

RethinkDB 是一個文件導向的資料庫軟體，每一筆資料皆是一個 JSON 文件。

一個伺服器下面會管理許多資料庫（Database），一個資料庫下面有許多資料表（table），資料表下面就會有許多文件（Document）。

這裡就不講解安裝與執行了。

使用 Node REPL 操作資料

我們先使用 RethinkDB 的 ReQL 以及 rethinkdb 套件來測試。

const r = require('rethinkdb')

使用 r 來建立指令，之後再傳送到伺服器上（送到 Socket），最後再由 Promise 回傳資訊。

建立連線

使用之前，一定要先建立一個可用的連線

r.connect('localhost:28015')
r.connect({
  // 若缺少任何值，該值將會使用預設值
  host: 'localhost',
  port: '28015',
  db: 'test',
  timeout: 20,
  authKey: '',
  ssl: {
    ca: caCert // Buffer Object
  }
})

兩個方法都會回傳 Promise。

var connection = null
r.connect()
 .then(established_connection => connection = established_connection)
 .catch(console.error)

關閉連線

使用之後，「理論上」應該要關閉連線。

手動關閉連線可以確保未獲得回應的要求順利完成（noreplyWait 選項）。

// 這一個方法會等待已經送出但還未收到回應的要求
connection.close().then(() => {
  // 若成功結束
}).catch(error => {
  // 若過程產生錯誤
})
// 這一個會嘗試立即關閉
const option = {
  noreplyWait: true
}
connection.close(option).then(() => {}).catch(error => {})

重新連線

// 這一個方法會等待已經送出但還未收到回應的要求
connection.reconnect().then(connection => {
  // 若重新連線成功
}).catch(error => {
  // 若過程產生錯誤
})
// 這一個會嘗試立即關閉
const option = {
  noreplyWait: true
}
connection.reconnect(option).then(connection => {}).catch(error => {})

在連線上選擇資料庫

若在要求中沒有選擇資料庫，將會使用連線預設的。

connecition.use('todo')

執行查詢

若已經透過 r 建立一個查詢（query），你可以透過連線執行它。

query.run(connection, {
  // 選項（可以省略）
}).then(result => {
  // result 的型態與 query 的種類有關
}).catch(error => {
  // 例外處理
})

RethinkDB 提供的選項有

  • readMode - 當在叢集中的讀取方式
    • single（預設值）- 從主伺服器（Primary replica）中讀取在記憶體中的內容（不一定被寫入硬碟）
    • majority - 從多個伺服器中，確認寫入硬碟中的資料大部分為哪寫（可能有部分伺服器尚未更新），並讀取
    • outdated - 從一個伺服器中讀取在記憶體中的內容（最快、最不準確）

  • timeFormat - 時間的格式

    • native (預設值) - 將會回傳成 Date 物件

    • raw - 回傳 JSON 表示式

      {  

        '$reql_type$': 'TIME',

        epoch_time: 1458120984.138,

        timezone: '+00:00'

      }

  • profile - 是否回傳查詢的效能資訊，預設值為 false

    { 

      profile: [

        {

          description: 'Evaluating table.',

          'duration(ms)': 0.019464,

          sub_tasks: [Object]

        },

        {

          description: 'Perform read.',

          'duration(ms)': 1.53457,

          sub_tasks: [Object]

        }

      ],

      value: Cursor {...} 

    }
  • durability
    • hard - 伺服器收到資料之後必須寫入，才會向客戶端送出回應
    • soft - 伺服器收到資料之後就會向客戶端送出回應
  • groupFormat - grouped_data 以及 grouped_stream 的格式
    • native
    • raw
  • noreply - 是否要回傳資料（若設為 false，就不需要處理常式）
  • db - 要查詢的資料庫（可以代替 r.db 或是 conn.use）
  • arrayLimit - 回傳的陣列大小上限。（預設值：100,000／註記：陣列寫入上限永遠是 100,000）
  • binaryFormat - 回傳二進位資料的方法
    • native - 回傳二進位資料
    • raw - 回傳存在資料庫中的 JSON 值（不進行轉換，跟 timeFormat 類似）
  • minBatchRows
  • maxBatchRows
  • maxBatchBytes
  • maxBatchSeconds
  • firstBatchScaleDownFactor

觀察變化

RethinkDB 提供連續觀察變化的 API，可以持續追蹤一個查詢的變化。

query.changes().then(cursor => {
  cursor.each(change => {
    // 對 change 做事情
    // 這個才是真正的 callback 函數
    // 若要停止觀察，直接 cursor.close() 即可
  })
})

changes() 也提供幾個選項，可以當作設定物件傳入 changes() 中。

  • squash - 設定若有一筆資料有多個變化，將會如何被送達
    • true - 多個變化會被合併成一個，使用戶端可以直接更新到最新資料
    • false（預設值）- 多個變化不會被合併
    • n - n 秒內的變化會被合併
  • changefeedQueueSize - 伺服器願意暫存的變化數量（預設值：100,000）
  • includeInitial - 是否傳送初始資料
  • includeStates - 是否在變化中包含 state 值（預設值：false）
    • state 包含 initializing 以及 ready

等待不被等待的查詢

若在查詢的時候將 noreplyWait 設為 true，但是必須在特定的時間等查詢完成，可以利用這個函數

connection.noreplyWait().then(() => {
  // 所以查詢都已經完成
}).catch(error => {
  // 發生錯誤
})

伺服器

可以透過連線物件取得伺服器的 UUID 以及名稱。

connection.server.then(server =>{
  // 伺服器是一個物件，其中有 id 以及 name 值
}).catch(error => {
  // 發生錯誤
})

連線的事件

連線物件本身會發出幾個事件

  • connect
  • close
  • timeout
  • error
const event = 'connect'
connection.addListener(event, () => {
  // 處理連線
})
connection.on(event, () => {
  
})
connection.once(event, () => {
  
})
connection.once(event, listener)
connection.removeAllListeners(event)
connection.setMaxListeners(n)
connection.listeners(event)
connection.emit(event, args, ...)

游標物件

從資料庫載入的資料以游標表示（而不是一個肥大的陣列）

游標的尋訪

可以透過 cursor.next, cursor.each, cursor.eachAsync 或是 cursor.toArray 來尋訪游標所指向的資料。

cursor.next().then(data => {
  // 下一筆資料
}).catch(error => {
  // 發生錯誤
})
cursor.each(data => {
  // 下一筆資料
}, () => {
  // 全部載入完成後的動作
})
cursor.eachAsync(data => {
  // 下一筆資料
}).then(() => {
  // 全部載入完成後的動作
}).catch(error =) {
  // 發生錯誤
})
cursor.toArray().then(array => {
  // 載入資料後的陣列
}).catch(error => {
  // 發生錯誤
})

關閉

若 cursor 還未完成讀取就被放棄，應該要 cursor.close()
