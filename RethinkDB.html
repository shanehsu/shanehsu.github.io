RethinkDB

RethinkDB

RethinkDB 是一個文件導向的資料庫軟體，每一筆資料皆是一個 JSON 文件。

一個伺服器下面會管理許多資料庫（Database），一個資料庫下面有許多資料表（table），資料表下面就會有許多文件（Document）。

這裡就不講解安裝與執行了。

使用 Node REPL 操作資料

我們先使用 RethinkDB 的 ReQL 以及 rethinkdb 套件來測試。

const r = require('rethinkdb')

使用 r 來建立指令，之後再傳送到伺服器上（送到 Socket），最後再由 Promise 回傳資訊。

建立連線

使用之前，一定要先建立一個可用的連線

r.connect('localhost:28015')
r.connect({
  // 若缺少任何值，該值將會使用預設值
  host: 'localhost',
  port: '28015',
  db: 'test',
  timeout: 20,
  authKey: '',
  ssl: {
    ca: caCert // Buffer Object
  }
})

兩個方法都會回傳 Promise。

var connection = null
r.connect()
 .then(established_connection => connection = established_connection)
 .catch(console.error)

關閉連線

使用之後，「理論上」應該要關閉連線。

手動關閉連線可以確保未獲得回應的要求順利完成（noreplyWait 選項）。

// 這一個方法會等待已經送出但還未收到回應的要求
connection.close().then(() => {
  // 若成功結束
}).catch(error => {
  // 若過程產生錯誤
})
// 這一個會嘗試立即關閉
const option = {
  noreplyWait: true
}
connection.close(option).then(() => {}).catch(error => {})

重新連線

// 這一個方法會等待已經送出但還未收到回應的要求
connection.reconnect().then(connection => {
  // 若重新連線成功
}).catch(error => {
  // 若過程產生錯誤
})
// 這一個會嘗試立即關閉
const option = {
  noreplyWait: true
}
connection.reconnect(option).then(connection => {}).catch(error => {})

在連線上選擇資料庫

若在要求中沒有選擇資料庫，將會使用連線預設的。

connecition.use('todo')

執行查詢

若已經透過 r 建立一個查詢（query），你可以透過連線執行它。

query.run(connection, {
  // 選項（可以省略）
}).then(result => {
  // result 的型態與 query 的種類有關
}).catch(error => {
  // 例外處理
})
