Node.js

套件系統

Node.js 有他自己的 npm 套件系統，以及他自己的 require() 系統，類似 Python 的 pip 以及 import。

package.json

除了小程式之外，你的專案資料夾應該先執行一次 npm init 建立 package.json 檔案。這個檔案會紀錄你專案所需要的套件以及當執行 npm start 所要執行的指令。因為有這個檔案的緣故，你在安裝所需的套件，應該使用 --save 參數

npm install express --save

這樣 npm 會自動幫你更新 package.json 檔案。

require() 系統

這個系統跟 ECMAScript 6 的模組類似，但是用法不一樣。

// circle.js
exports.area = (r) => Math.PI * r * r
exports.circumference = (r) => 2 * Math.PI * r
const circle = require('./circle.js')
circle.area(4)

因為我們這次 require 的是同一個目錄的檔案，若是要 require 從 npm 取得的套件

const express = require('express')

若是要把自己的程式碼包裝成套件，可以將所有的 exports 寫在 index.js 檔案之中，存在一個資料夾內，之後用 require('./folder')。

當然，若該資料夾包含一個 package.json 檔案，你可以透過更改 main 鍵值，重新命名主要模組的名稱。

子程序

若真的有需要直接利用指令，內建的 child_process 模組非常的好用。基本用法如下

const spawn = require('child_process').spawn;
const process = spawn('ls', ['-lh', '/usr'])
process.stdout.on('data', data => console.log(`stdout: ${data}`))
process.stderr.on('data', data => console.error(`stdout: ${data}`))
process.on('close', code => console.log(`Process exited with code ${code}`))

事件

非同步處理是 Node.js 中非常重要的一環，幾乎所有的事情都需要等待（等待資料庫、等待連線），所以用事件來驅動處理常式是解決方案。

const EventEmitter = reuqire('events')
const util = require('util')
const emitter = new EventEmitter()
emitter.on('event', () => console.log('事件發生了。'))
emitter.emit('event')
